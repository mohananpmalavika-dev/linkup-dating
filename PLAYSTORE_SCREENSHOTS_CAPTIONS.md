Your app screenshots are the MOST IMPORTANT part of your store listing. They appear before the description and heavily influence download decisions.

**Key Stats:**
- 🎯 Screenshots influence 80% of download decisions

- 📱 Must be 1080×1920 pixels (9:16 aspect ratio)
- 🎨 Use app's actual UI (not mockups)
- ✍️ Keep captions short & punchy (15-20 words max)

## 📝 CAPTION BEST PRACTICES

✅ **DO:**
- Keep under 20 words per caption
- Use action verbs (Meet, Swipe, Chat, Find)
- Highlight benefits, not features
- Be positive and inviting
- Use emojis sparingly (1-2 max per caption)
- Show real app interface (not stock photos)
- Each screenshot should tell part of story

❌ **DON'T:**
- Make captions too long
- Use technical jargon
- Be misleading or exaggerate
- Show fake/stock images
- Use all caps (seems aggressive)
- Include personal information
- Repeat same caption twice

## 🖼️ SCREENSHOT SPECIFICATIONS

### Required Format:
| Property | Value |
|---|---|
| Resolution | 1080 × 1920 pixels |
| Aspect Ratio | 9:16 |
| File Format | PNG or JPEG |
| Max File Size | 8 MB per image |
| Quantity | 2-8 images (recommend 5) |
| DPI | 72+ DPI |

### Common Mistakes to Avoid:
- ❌ Portrait mode (use landscape if needed - will auto-rotate)
- ❌ Too small resolution (must be exactly 1080×1920)
- ❌ Including device bezel/navigation bar
- ❌ Watermarks or copyright text
- ❌ Blurry or low-quality images
- ❌ Stock photos instead of app screenshots

### Image Optimization:
```bash
# If your screenshots are wrong size, resize using ImageMagick:
magick screenshot1.jpg -resize 1080x1920 screenshot1-resized.jpg

# Or use online tool: https://ezgif.com/
# Or Photoshop/GIMP
```
