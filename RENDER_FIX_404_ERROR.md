# 🔧 Fix Render Deployment - 404 Error Solution

## ⚠️ Current Issue

Your frontend shows:
```
Failed to load resource: /api/v1/mci 404
TypeError: Cannot read properties...
```

**Cause:** Environment variables not set in Render, API endpoints wrong

---

## ✅ Fix in 5 Steps

### Step 1: Update Frontend Environment on Render

1. Go to: https://render.com/dashboard
2. Click **"linkup-frontend"** service
3. Click **"Environment"** tab (or "Env" on left)
4. Look for these variables:
   ```
   REACT_APP_API_URL
   REACT_APP_BACKEND_URL
   ```

5. If they exist, update them:
   ```
   REACT_APP_API_URL = https://linkup-backend.onrender.com/api
   REACT_APP_BACKEND_URL = https://linkup-backend.onrender.com
   ```

6. If they don't exist, click **"Add Environment Variable"**:
   - First variable:
     - Key: `REACT_APP_API_URL`
     - Value: `https://linkup-backend.onrender.com/api`
     - Click "Save"
   
   - Second variable:
     - Key: `REACT_APP_BACKEND_URL`
     - Value: `https://linkup-backend.onrender.com`
     - Click "Save"

7. Wait for **auto-redeploy** (might take 5 minutes)

---

### Step 2: Update Backend Environment on Render

1. Click **"linkup-backend"** service
2. Click **"Environment"** tab
3. Verify these exist:
   ```
   DATABASE_URL: [should be auto-filled from PostgreSQL]
   JWT_SECRET: [should be set]
   FRONTEND_URL: https://linkup-frontend.onrender.com
   NODE_ENV: production
   ```

4. If `DATABASE_URL` is missing:
   - Go to your PostgreSQL database service
   - Copy the "Internal Database URL"
   - Add it to backend environment variables

5. Click **"Redeploy"** button (top) to restart with new vars

---

### Step 3: Clear Browser Cache

1. Open Frontend: https://linkup-frontend.onrender.com
2. Press:
   - **Windows:** Ctrl + Shift + Delete
   - **Mac:** Cmd + Shift + Delete
3. Clear **All Time**
4. Hard refresh:
   - **Windows:** Ctrl + F5
   - **Mac:** Cmd + Shift + R

---

### Step 4: Check Backend is Running

```bash
# In terminal, test backend:
curl https://linkup-backend.onrender.com/health

# Should return:
# {
#   "status": "Backend is running",
#   "timestamp": "2026-04-26T..."
# }
```

If this fails:
1. Render Dashboard → Backend Service
2. Click **"Logs"** tab
3. Look for error messages
4. Likely issues:
   - Database not connected
   - Build failed
   - JWT_SECRET not set

---

### Step 5: Test Again

1. Open: https://linkup-frontend.onrender.com
2. Press **F12** to open DevTools
3. Go to **Console** tab
4. Should see **no 404 errors**
5. Try **Sign Up** - should work now!

## 🔍 Debug: Check What Went Wrong

### If Backend Health Check Fails

Open Render Backend Logs:
1. Render Dashboard → **linkup-backend** service
2. Click **"Logs"** tab (bottom)
3. Look for:
   - ✓ "✓ Connected to PostgreSQL" → DB working
   - ✓ "🚀 Dating app backend running" → Server started
   - ✗ "Error: connect ECONNREFUSED" → Database not connected
   - ✗ "Error: Cannot find module" → Missing dependency

**If you see errors:**
1. Click **"Redeploy"** button (top)
2. Wait for redeployment to complete
3. Check logs again

---

### If Frontend Shows Blank

1. Open DevTools (F12)
2. Go to **Network** tab
3. Look for failed requests
4. Common issues:
   - API_URL wrong → Check environment variables
   - Backend not running → Check backend logs
   - Build failed → Render would show "Build Error"

## 🆘 Still Having Issues?

### Enable Debug Logging

**Backend:**
1. Render Dashboard → Backend Service
2. Environment → Add:
   ```
   LOG_LEVEL=debug
   ```

**Frontend:**
1. Open DevTools (F12)
2. Console should show all API calls
3. Check what URL it's actually calling

### Common Mistakes

1. ❌ Missing `https://` in URL
2. ❌ Typo in domain name (onrender.com not onrender.io)
3. ❌ Copy-paste added extra spaces
4. ❌ Forgot to save after editing environment variable
5. ❌ Using old localhost:5000 URLs
